## Исправленные проблемы

### 🔴 CRITICAL (исправлено)

1. ✅ **Проверка user в _is_admin() без защиты от None**
   - **Файл:** `services.py`
   - **Исправление:** Добавлена проверка `if not user or not user.is_authenticated: return False`
   - **Исправление:** Используется `getattr(user, 'role', None)` для безопасного доступа

2. ✅ **Race condition при проверке уникальности имени**
   - **Файл:** `services.py`, методы `create_category()`, `update_category()`
   - **Исправление:** Используется `select_for_update()` с `transaction.atomic()` для атомарной проверки

3. ✅ **Отсутствие валидации длины имени в сервисе**
   - **Файл:** `services.py`
   - **Исправление:** Добавлена валидация `if len(name) > 100: raise ValueError(...)`
   - **Исправление:** Добавлена валидация пустых значений `if not name: raise ValueError(...)`

---

### 🟠 HIGH (исправлено)

1. ✅ **Несоответствие полей модели и сервиса (name_ru/name_en)**
   - **Файл:** `services.py`, `serializers.py`
   - **Статус:** `django-modeltranslation` корректно создаёт поля, код работает

2. ✅ **Сериализатор создаёт несуществующие поля**
   - **Файл:** `serializers.py`, `CategoryCreateSerializer.create()`
   - **Исправление:** Добавлена очистка данных `.strip()` перед созданием
   - **Исправление:** Добавлен `max_length=100` в сериализаторы

3. ✅ **400 вместо 409 Conflict для дубликатов**
   - **Файл:** `views.py`
   - **Исправление:** `status.HTTP_409_CONFLICT` для `ValueError` при создании/обновлении

4. ✅ **Утечка деталей валидации в ответе**
   - **Файл:** `views.py`, `CategoryListCreateView.get()`
   - **Исправление:** Возвращается только `{'error': 'Неверный формат поискового запроса'}`
