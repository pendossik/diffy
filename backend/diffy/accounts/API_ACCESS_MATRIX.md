## 🔐 Уровни доступа

| Значение | Описание |
|----------|----------|
| ✅ **Все** | Доступно всем (включая неавторизованных) |
| 🔑 **IsAuthenticated** | Требуется аутентификация (любой пользователь) |
| 🔒 **IsAdmin** | Требуется аутентификация + роль `admin` или `superuser` |

## ⚠️ Важные замечания

1. **Двухуровневая защита:**
   - Уровень 1: `get_permissions()` в views (AllowAny / IsAuthenticated)
   - Уровень 2: `_is_admin()` в сервисах (проверка роли)

2. **Для GET запросов:**
   - Проверка `_is_admin()` **не требуется**
   - Доступно всем пользователям (включая неавторизованных)

3. **Для POST/PUT/PATCH/DELETE:**
   - Требуется аутентификация (`IsAuthenticated`)
   - Требуется роль администратора (`_is_admin()`)
   - Без обеих проверок операция будет отклонена

4. **Роли пользователей:**
   - `user` — обычный пользователь (только чтение)
   - `admin` — администратор (чтение + запись)
   - `superuser` — суперпользователь (чтение + запись + все права)
