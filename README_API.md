## 📘 Документация API (Swagger)

Для просмотра и тестирования API вашего проекта используйте интерактивную документацию Swagger UI.

### 🔗 Как получить доступ к Swagger UI

1. Запустите backend-сервер:
   ```bash
   python manage.py runserver
   ```

2. Перейдите в браузере по адресу:
   ```
   http://localhost:8000/api/docs/
   ```

3. Вы увидите интерактивный интерфейс, где можно:
   - Просматривать все доступные эндпоинты.
   - Смотреть описание каждого метода (summary, tags, параметры).
   - Тестировать запросы прямо из браузера (включая авторизацию через JWT токены).

### 📥 Скачать OpenAPI схему (YAML)

Если вам нужна схема для генерации клиентских библиотек или интеграции с другими инструментами, скачайте её по ссылке:

```
http://localhost:8000/api/schema/
```

Файл будет в формате YAML — стандартном для OpenAPI Specification (OAS).

### ⚙️ Примечания

- Для работы с защищенными эндпоинтами (`/api/accounts/current_user/`, `/api/compare/favorites/`) необходимо сначала получить JWT токен через `/api/token/`.
- В Swagger UI вы можете использовать кнопку **"Authorize"** в правом верхнем углу, чтобы ввести токен и выполнять аутентифицированные запросы.
- Также доступен альтернативный интерфейс Redoc по адресу: `http://localhost:8000/api/redoc/`.
